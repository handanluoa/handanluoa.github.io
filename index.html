Overview

项目概览

项目目标

构建一个面向视频生成与偏好学习的数据系统，让模型不仅学会视频内容生成， 还能够学会构图、光线、色彩、人物气质和镜头节奏等稳定的视觉风格偏置。

核心问题

普通视频数据常见的问题是文本脏乱、风格混杂、人物 identity 漂移、时序信号不稳定。 本项目通过受控词表、镜头级切片、结构化审核和偏好对比数据来解决这些问题。

落地价值

这套系统既能服务训练数据生产，也能作为后续 reward modeling、DPO、 reranking 与审美 benchmark 的基础设施。

System Design

系统设计

01 / DATA LAYERS

四层数据结构

以 S 级锚点数据为核心，配合 A 级主体扩展、B 级长尾补充和 P 级偏好样本， 让训练既保持风格纯度，又能避免模板化。

  • S: 核心审美锚点
  • A: 高质量扩展样本
  • B: 稀缺分布补洞
  • P: chosen / rejected 偏好对

02 / STYLE SYSTEM

受控风格桶

首版聚焦三个主风格桶，避免风格中心过散，并为每个桶建立稳定的光线、色调、 构图和动作表达方式。

  • `minimal_indoor_portrait`
  • `soft_outdoor_lifestyle`
  • `cinematic_urban_closeup`

03 / TEXT SUPERVISION

三层文本监督

每个 clip 同时生成 factual caption、aesthetic caption 与 training prompt， 分别负责强对齐、显式审美表达与训练推理迁移。

  • 事实层: 只写可见信息
  • 审美层: 描述气质与风格
  • 生成层: 压缩成可控 prompt

04 / IDENTITY

人物一致性治理

通过 face clustering、人工复核和 reference 图集建设，为同一人物分配稳定 identity_id， 避免模型把服装或场景误学为人物特征。

Schema

Workflow

数据生产与审核工作流

Step 1

采集与预清洗

记录来源、授权、原始视频信息，并去除重复素材、强字幕、水印与明显偏离目标审美的片段。

Step 2

镜头级切片

将长视频切为 2-8 秒 clip，每条样本尽量只承载一个主主体、一个主动作和一种主要镜头语言。

Step 3

自动分析与文本重写

生成初始 metadata，再按照标注规范重写 factual caption、aesthetic caption 与 training prompt。

Step 4

identity 归并与质检

做人脸聚类和人工校验，检查文本对齐、审美纯度、人物一致性，再给出 S / A / B / P 等级。

Step 5

偏好学习样本构建

对语义相近样本进行 chosen / rejected 比较，记录偏好原因，为后续 DPO 或 reward modeling 提供基础。

Roadmap

下一步计划

MVP

先围绕 3 个 aesthetic buckets 构建 300-1000 个高质量 clip，验证风格迁移与 identity 稳定性。

Training

先保留基础视频生成能力，再注入审美偏好数据，最后使用偏好样本进行进一步优化。

Evaluation

重点观察 prompt 风格可迁移性、输出审美偏置、人物一致性和模板化风险。
